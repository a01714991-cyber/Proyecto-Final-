Este proyecto final implementa un sistema de tienda en C++ basado en los principios fundamentales de la Programación Orientada a Objetos. El sistema modela distintos tipos de productos, específicamente playeras, shorts deportivos y accesorios, los cuales heredan de una clase base abstracta llamada Producto. Esta estructura permite que cada clase hija implemente sus propios atributos y comportamientos mediante el uso de funciones virtuales y métodos sobrescritos, manteniendo así un diseño modular, escalable y coherente con las buenas prácticas de herencia.

Para garantizar un manejo adecuado de la información, se aplicó encapsulación mediante el uso de atributos privados y protegidos dentro de cada clase. Asimismo, se incorporó una clase Inventario que administra una colección de objetos Producto mediante agregación, utilizando un vector que almacena punteros a objetos creados dinámicamente por el usuario. Esta relación está representada también en el diagrama UML donde Inventario contiene un conjunto de productos sin poseerlos completamente, lo cual coincide con el comportamiento implementado en el código.

Además de la estructura orientada a objetos, se revisaron la congruencia y consistencia entre el diagrama de clases, el código fuente y el archivo README mostrado en este repositorio. Se realizaron correcciones en el diagrama UML para incluir herencia, agregar la agregación entre Inventario y Producto, y verificar que los atributos y métodos definidos coincidieran con la implementación real. También se ajustaron nombres, niveles de acceso y definición de métodos para asegurar que la lógica del programa correspondiera con el diseño planificado.

En este proyecto también se añadieron de forma explícita las instrucciones necesarias para compilar y ejecutar el programa. Para sistemas macOS o Linux es suficiente con utilizar un compilador compatible con C++11 o superior y ejecutar el comando g++ tienda.cpp -o tienda, seguido de ./tienda. En Windows se puede compilar empleando MinGW o un entorno equivalente que admita la misma versión del estándar. El programa pide al usuario la información necesaria para crear cada uno de los productos, los agrega al inventario y posteriormente muestra toda la información capturada, permitiendo visualizar la funcionalidad completa del sistema.

Asi mismo, se actualizaron los apartados de correcciones solicitados en la rúbrica del proyecto. Se documentó qué modificaciones se realizaron, a qué indicador o subcompetencia pertenecen y en dónde se encuentran dentro del repositorio. Esto facilita la revisión del profesor y asegura que el proyecto cumpla con las expectativas planteadas en la evaluación final. Finalmente, todo el sistema es compatible con cualquier entorno que disponga de un compilador C++ moderno y no requiere librerías externas adicionales, lo que permite su portabilidad y ejecución sin configuraciones adicionales.
